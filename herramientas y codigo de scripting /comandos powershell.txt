 Comando Principal de Análisis (Lab #2)

Este comando filtra procesos específicos basándose en su uso de memoria:

powershell
Get-Process -Name "Code" | Where-Object { $_.WorkingSet -gt 200MB } | Select-Object Id, ProcessName


### **Instrucciones para el Lab #1**

1. En el CMD, ejecutar:
cmd
tasklist


*Luego, elija un proceso de la lista y copie su nombre (sin la extensión .exe).*
2. **En PowerShell, ejecutar el filtro:**
powershell
Get-Process -Name "nombre-proceso" | Where-Object { $_.WorkingSet -gt valor-numerico-unidad-almacenamiento }


> Nota: Sustituye `nombre-proceso` por el nombre que elegiste y `valor-numerico-unidad-almacenamiento` por un valor (ejemplo: `100MB`).

 1. Filtrar por Uso de CPU (Carga de trabajo)

Si quieres encontrar procesos que están consumiendo mucho procesador en lugar de memoria:

```powershell
Get-Process | Where-Object { $_.CPU -gt 50 } | Select-Object Id, ProcessName, CPU

```

* **Explicación:** Muestra procesos que han usado más de 50 segundos de tiempo de CPU.

### 2. Filtrar por Compañía o Fabricante

Útil para identificar procesos que no son de Microsoft o de un proveedor específico:

```powershell
Get-Process | Where-Object { $_.Company -notlike "*Microsoft*" } | Select-Object ProcessName, Company

```

* **Explicación:** Filtra y muestra solo procesos cuyo fabricante **no** sea Microsoft.

### 3. Ordenar los resultados (Sorting)

Puedes combinar el filtro de memoria que ya tienes con un ordenamiento para ver quién consume más arriba:

```powershell
Get-Process | Where-Object { $_.WorkingSet -gt 100MB } | Sort-Object WorkingSet -Descending | Select-Object Id, ProcessName, WorkingSet



* **Explicación:** Filtra procesos mayores a 100MB y los ordena de mayor a menor consumo.

### 4. Filtrar por múltiples condiciones (Operadores Lógicos)

Puedes buscar procesos que cumplan dos cosas a la vez usando `-and`:

```powershell
Get-Process | Where-Object { $_.WorkingSet -gt 100MB -and $_.ProcessName -like "s*" }


* **Explicación:** Busca procesos que pesen más de 100MB **y** cuyo nombre empiece con la letra "s".

### 5. Contar cuántos procesos cumplen el filtro

Si solo necesitas el número total para un reporte:

```powershell
(Get-Process | Where-Object { $_.WorkingSet -gt 200MB }).Count
